# Python deps bundled into the packaged installer via scripts/bundle-python-main.ps1.
#
# Derived from backend/requirements.txt MINUS packages that don't have
# prebuilt cp311 win_amd64 wheels (so they can't be bundled into a Python
# 3.11.9 embeddable). Currently excluded:
#
#   - hnswlib>=0.8.0  (ships source-only on Windows; bundling requires
#     numpy + pybind11 + cython + Visual Studio C++ toolchain at CI time —
#     see https://github.com/nmslib/hnswlib/issues — and adds ~2 minutes
#     to release CI for an optional vector store backend)
#
# `import backend.main` (the post-install verify step in
# scripts/bundle-python-main.ps1) does NOT transitively import hnswlib, so
# the bundled Python can satisfy the launch-time `import backend.main` even
# without hnswlib installed. Users hitting the wiki/vectorstore_hnsw.py
# code path will see a runtime ImportError if they don't have hnswlib on
# their system (this is the same trade-off release/win7 LTS has lived
# with since v0.4.0-lts and which main now inherits — there is no fix
# upstream).
#
# This file should match backend/requirements.txt byte-for-byte minus the
# hnswlib line (and any other future source-only packages). Diff with:
#
#   diff backend/requirements.txt backend/requirements-bundled.txt
#
# If you add a new dep to requirements.txt, decide whether it needs bundling
# (most pure-Python / wheel-shippers: yes) or runtime-install only
# (anything with a C extension and no win wheel: put in
# requirements-bundled.txt's excluded list comment, not in this file).
