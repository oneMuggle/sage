# 07. 自定义标题栏与前进后退 (Custom Titlebar & Navigation)

**最后更新**:2026-06-29
**适用版本**:Sage v0.1+ (含 M9 + M7 升级, win7 `v*-lts` 与 main `v*` 均适用)

## 7.1 概览

Sage 的桌面窗口顶部有一行**自定义标题栏**,替代操作系统默认的标题栏:

- 左侧:**← (后退) / → (前进)** 按钮,跨路由前进/后退
- 右侧(Windows / Linux):**最小化 / 最大化 / 关闭** 窗口控制三按钮

> Web 模式(浏览器中运行)只显示左右两按钮,没有窗口控制;macOS 用系统自带的红黄绿 traffic lights。

## 7.2 前进 / 后退按钮

### 它是什么

浏览器风格 ↩ ↪ 按钮,内置在窗口顶部左侧。无论你刚才访问的是哪个页面
(`/chat` → `/welcome` → `/settings`),都可以通过它一步退回或前进。

### 操作步骤

1. 鼠标移到顶部标题栏**最左侧**的 ↩(向左)或 →(向右)按钮
2. 单击即可

### 行为细节

- **按钮状态**:
  - ↩ 在栈底(已经到最早可访问的页面)时**禁用**,灰显,鼠标光标呈 not-allowed
  - → 在栈顶(已经到最新页面)时**禁用**,灰显
- **栈容量**:`MAX_HISTORY = 50`,超出后最早的访问会被丢弃
- **导航与浏览器后退的关系**:每次点击 ↩/→ 同样会触发 React Router `navigate(-1)/(1)`,
  页面内容切换 + 标题栏按钮状态同步更新
- **持久范围**:仅本次会话,不写入本地存储;重启 Sage 后栈清空

### 何时用

- 从欢迎屏(`/welcome`)误跳到设置,想快速撤回 → ↩
- 浏览设置过程,想"撤销"我刚刚看的页面 → ↩
- 看了设置,又跳回聊天 → →

## 7.3 窗口控制按钮(Windows / Linux)

### 它是什么

桌面端窗口右上角的三个按钮,用于最小化、最大化、关闭 Sage。

### 操作步骤

1. 鼠标移到顶部标题栏**最右侧**的按钮区
2. 单击对应按钮:
   - **`−`(最小化)**:把窗口缩到任务栏
   - **`▢`(最大化)** 或 **`❐`(还原)**:在最大化与窗口模式间切换
   - **`×`(关闭)**:直接退出 Sage

> **悬停效果**:`×` 按钮悬停时背景变红,提示这是"危险"操作。
> 三个按钮的 hover 颜色都来自主题,与 Sage 主色保持一致。

### 行为细节

- **最小化 / 关闭**后,再次启动 Sage 不会保留之前的会话状态
  (除非系统级自动恢复)
- **最大化**是 toggle 模式:再次点击会还原为原始窗口大小
- **关闭**:与点击系统标题栏的关闭一致,会触发 `beforeunload` 清理流程

### 何时用

- 需要把 Sage 临时收起去做别的事 → `−`
- 需要全屏看聊天 / 协调看板的内容 → `▢`
- 不用时 → `×`

## 7.4 平台差异速查

| 平台 | 后退/前进 | 窗口控制 |
|------|----------|----------|
| Windows 10/11 | ✅ ↩ / → | ✅ `−` / `▢` / `×` |
| Windows 7 SP1 (LTS) | ✅ ↩ / → | ✅ `−` / `▢` / `×`(与 Win10/11 一致) |
| Linux (Ubuntu 等) | ✅ ↩ / → | ✅ `−` / `▢` / `×` |
| macOS | ✅ ↩ / → | ❌(系统原生 traffic lights 替代) |
| Web 模式(浏览器) | ✅ ↩ / → | ❌(无窗口概念) |

## 7.5 FAQ

### 我点了 ↩ 但页面没变?

- 可能已经在栈底,按钮已禁用。
- 也可能刚刚通过 Sidebar + 新对话按钮导航到 `/welcome`,该操作是 Navigate 而非 back,
  不计入 NavHistory 栈。

### 我把窗口拖到顶部边缘会自动最大化吗?

- **会**。Windows 10/11 与 macOS 的"边缘吸顶"特性是操作系统提供的,Sage 没主动关闭它。
- 想要手动恢复:再点一下最大化按钮(此时已变成还原图标 `❐`)即可。

### 在 Windows 7 上拖动标题栏没反应?

- 这是个**已知问题**,已通过 `drag` CSS 类修复(详见
  `docs/technical/28-phase5-titlebar.md` 的 drag region 章节)。
- 如果还有问题,请在 [GitHub Issues](https://github.com/oneMuggle/sage/issues) 报告。

### 后退按钮经常被窗口拖动吃掉,按不到怎么办?

- 标题栏外层是 `drag` class(`-webkit-app-region: drag`),后
  退/前进按钮的外层 div 专门加了 `no-drag`(`-webkit-app-region: no-drag`)
  ,所以按钮点击不会被 drag 吞掉。
- 如果你环境里看到相反情况(按钮按了没反应,反而在拖动窗口),可能是 CSS 没
  加载。建议清缓存 / 重启 Sage。
