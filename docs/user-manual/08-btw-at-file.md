# M8 /btw 补充消息 + @文件提及 — 用户手册

## 什么是 /btw 补充消息?

在主对话运行中(LLM 正在生成答案),你可以同时提一个"顺便问"型问题。例如:

```
主对话: "帮我写一段 Python 异步爬虫"
LLM: 正在生成...
你:    /btw 那 requests 和 aiohttp 哪个更适合这个场景?
```

`/btw` 命令会在屏幕角落弹出一个浮层,LLM 单独处理这个问题,主对话完全不受影响。等浮层显示答案后,你可以继续看主对话。

## 什么是 @文件提及?

在输入消息时,敲 `@` 触发工作区文件搜索菜单。例如:

```
你: 请帮我看看 @src/utils/parser.
                          ^ 文件搜索菜单弹出
                          ↓ 输入 "ts" 过滤
                          ↓ 看到 src/utils/parser.ts
                          ↓ Enter 选中
                          ↓ 文本变为 @src/utils/parser.ts
```

选中后,文件路径会作为上下文传给 LLM,LLM 可以直接读这个文件。

## 操作步骤

### /btw 触发

1. 在主对话输入框中,输入 `/btw ` 加上你的问题
2. 浮层自动在屏幕右下角弹出,显示 "思考中..."
3. LLM 开始流式回答,边想边显示
4. 等答案完整后,可以:
   - 关闭浮层(× 按钮或 Esc)
   - 继续问下一个 /btw
   - 回到主对话

### /btw 多实例

- 第二次触发 /btw,会自动关闭前一个 overlay
- 加载中按 Esc:只关闭 overlay,不取消主请求

### @文件提及触发

1. 在输入框中敲 `@`,触发文件搜索菜单
2. 继续输入文件名片段(支持模糊搜索),菜单实时过滤
3. 用 ↑↓ 键选择,Enter 确认
4. 选中后,文件路径插入到光标位置,后续输入继续

### @文件提及超时处理

- 文件搜索 3 秒超时,菜单显示"搜索超时"
- 点击"重试"按钮重新搜索

## 快捷键

| 键 | 功能 |
|----|------|
| `/btw ` + 问题 | 触发补充消息面板 |
| `@` + 文件名片段 | 触发文件搜索菜单 |
| `↑` / `↓` | 在文件菜单中切换 |
| `Enter` | 选中当前项 |
| `Esc` | 关闭浮层(不取消请求) |

## 常见问题

### Q: /btw 触发了但浮层没出来?
A: 检查输入格式 — 必须是 `/btw ` (含空格) + 问题。例如 `/btw 1+1=?` 不会触发,因为前面没有空格。

### Q: @ 文件搜索一直显示"搜索超时"?
A: 工作区文件很多时,搜索会变慢。可以:
- 输入更精确的文件名片段
- 等待 3 秒后点击"重试"

### Q: /btw 答案和主对话答案会互相影响吗?
A: 不会。/btw 是独立的 LLM 调用,不修改主对话的 messages。但你可以手动把 /btw 答案复制到主对话。

### Q: 在主对话运行中能触发多个 /btw 吗?
A: 不能。第二次触发会关闭前一个,但前一个未读答案会丢失。如需保留,先关闭再读再触发。

## 技术细节(给开发者)

- /btw 走 `useChat.sendMessage(content, sessionId, { btw: BtwPayload })` 第三参数
- @ 文件搜索走 `fileSearchClient.search(query, { limit, signal })` 3s AbortController
- btwState 是 Zustand store,多组件共享(ChatInput 触发,MessageList 渲染 BtwOverlay)
- 所有 i18n 走 `useI18n()` hook,zh.ts / en.ts 双语支持
