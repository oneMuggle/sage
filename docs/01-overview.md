# Sage - 项目概述

## 1.1 项目背景

### 1.1.1 问题陈述

当前 Windows 7 用户面临以下困境：

1. **现代 AI 工具断供**
   - Chatbox 要求 Node.js ≥20.0.0，Electron 32+
   - Hermes Agent 基于 Python 3.10+，无桌面客户端
   - 主流 AI 助手均已放弃 Win7 支持

2. **轻量级需求被忽视**
   - Win7 机器通常配置较低
   - Electron 应用内存占用 150MB+
   - 需要更轻量的解决方案

3. **记忆与进化能力缺失**
   - 现有桌面 AI 助手多为单次对话
   - 无跨会话记忆能力
   - 无法从交互中学习和进化

### 1.1.2 目标用户

| 用户画像 | 需求 |
|---------|------|
| Win7 怀旧用户 | 继续使用熟悉系统的轻度 AI 助手 |
| 低配置电脑用户 | 需要轻量级 AI 工具 |
| 记忆型助手需求者 | 希望 AI 记住偏好和上下文 |
| 开发者/极客 | 喜欢可定制、可扩展的 AI 工具 |

---

## 1.2 项目定位

### 1.2.1 产品定义

**Sage** 是一款专为 Windows 7 系统打造的轻量级 AI 桌面助手，具备持久记忆系统和自我进化能力。

### 1.2.2 核心价值

| 价值 | 说明 |
|-----|------|
| **记忆型** | 跨会话记住用户偏好、习惯、重要信息 |
| **进化型** | 从交互中学习，不断优化响应质量 |
| **轻量型** | Win7 流畅运行，资源占用小 |
| **可扩展** | 支持技能插件和自定义配置 |

### 1.2.3 Slogan

> **"Sage - 你的记忆型 AI 助手"**

## 1.4 技术选型

### 1.4.1 核心技术栈

| 层级 | 技术选型 | 版本 | 说明 |
|-----|---------|------|------|
| 桌面框架 | Tauri | 1.x | Win7 兼容 |
| 前端框架 | React | 18.x | 成熟生态 |
| 前端构建 | Vite | 5.x | 快速构建 |
| 后端语言 | Python | 3.8-3.11 | 广泛兼容 |
| 数据库 | SQLite | 3.x | 零依赖 |
| 向量存储 | ChromaDB | 0.4.x | 轻量向量 DB |
| AI SDK | openai-python | 1.x | 统一接口 |
| HTTP 客户端 | httpx | 0.25.x | 异步支持 |
| ORM | SQLAlchemy | 2.x | 数据库抽象 |

### 1.4.2 技术选型理由

**为什么选择 Tauri 而不是 Electron？**

| 对比项 | Electron | Tauri 1.x |
|-------|----------|-----------|
| 内核 | Chromium (独立) | WebView2 (系统内置) |
| Win7 支持 | 已终止 | 支持 (需 KB4549949) |
| 内存占用 | ~150MB+ | ~20MB |
| 安装包 | ~100MB+ | ~10MB |
| Node 要求 | ≥20.0.0 | 无 (Rust 后端) |

**为什么选择 Python 后端？**

1. **灵活性** - 快速迭代，易于调试
2. **生态丰富** - AI 库、工具库完善
3. **学习成本低** - 用户可自行扩展
4. **跨平台** - Linux/Mac 也能运行

**为什么选择 ChromaDB？**

1. **轻量** - 单文件，易部署
2. **本地优先** - 无需云服务
3. **向量检索** - 支持语义记忆
4. **Python 原生** - 与后端无缝集成
