## 背景与目标

LLM Wiki 是一种个人知识库模式——LLM 持续构建和维护一个结构化的 wiki（markdown 文件集合），而非传统的 RAG 检索。核心思想是：知识库是一个**持续的、复利的产物**，每次导入新资料时，LLM 会提取关键信息并整合到现有 wiki 中，更新相关页面、修订摘要、标注矛盾。

## 架构设计

### 目录结构

```
wiki-project/
├── raw/
│   ├── sources/        # 原始文档（不可变）
│   └── assets/         # 附件资源
├── wiki/
│   ├── entities/       # 实体页面
│   ├── concepts/       # 概念页面
│   ├── sources/        # 源文档摘要页面
│   ├── queries/        # 查询结果页面
│   ├── overview.md     # 总览
│   ├── index.md        # 内容索引（自动维护）
│   └── log.md          # 活动日志（自动维护）
└── schema.md           # 结构配置文档
```

### 技术栈

- **文件 I/O**: Rust/Tauri 直接操作（不走 Python 代理）
- **LLM 调用**: 复用 Settings 中配置的端点和模型
- **搜索**: Rust 端全文搜索（CJK 分词）
- **前端**: Zustand store + React 组件

## 风险评估

| 风险                 | 级别 | 缓解措施                             |
| -------------------- | ---- | ------------------------------------ |
| 路径穿越漏洞         | 高   | 所有路径必须经过 canonicalize() 校验 |
| LLM 调用失败         | 中   | 重试 + 回退保存原始内容              |
| 大文件超出上下文窗口 | 中   | 检查文件大小，分块或拒绝             |
| 并发写入损坏文件     | 低   | 原子写入（临时文件 + rename）        |
