**Goal:** 修复 release/win7 分支 CI 上 `Tauri 1.6 build (cross-build, Win7 冒烟需人工)` job 失败 —— cargo 1.77.2 无法解析 `zeroize 1.9.0`（要求 `edition2024`，但 Rust 1.85 才稳定 edition2024）。

**Architecture:** 沿用 docs/technical/20-win7-tauri-compat.md §3.1 已有的「显式 cap edition2024-依赖」模式（hashbrown/ahash/indexmap/ignore/globset/tokio/chrono/uuid/futures-util），补一个之前漏掉的 `zeroize = "<1.9"` cap，让 Cargo.toml 在 Linux + Windows 两条 target 路径下都拒绝拉入 zeroize ≥ 1.9.0。

**Tech Stack:** Rust 1.77.2 (CI toolchain), Rust 1.96.0 (本地), Cargo, GitHub Actions, Tauri 2.1.1 + Win7 CVE backport fork, @tauri-apps/api 2.1.0.

## 上下文：实际失败点（4 次连续 CI run 失败）

**根因**（CI run `27452948313` log，2026-06-13 01:49 UTC）：

```
error: failed to parse manifest at
  `C:\Users\runneradmin\.cargo\registry\src\index.crates.io-...\zeroize-1.9.0\Cargo.toml`

Caused by:
  feature `edition2024` is required

  The package requires the Cargo feature called `edition2024`, but that feature
  is not stabilized in this version of Cargo (1.77.2 (e52e36006 2024-03-26)).
  Consider trying a newer version of Cargo (this may require the nightly release).

failed to build app: failed to build app
##[error]Process completed with exit code 1.
```

**根因分析**：

1. `release/win7` CI 装的是 `dtolnay/rust-toolchain@1.77.2`（Win7 兼容必须 1.77.2，1.78+ 移除 Win7 一级支持）。
2. CI `npx tauri build --bundles msi,nsis` 走 **Windows target**（msi/nsis 是 Windows 专属 bundle），cargo 在解析 Windows 路径时拉入了 `zeroize 1.9.0`（edition2024）作为某个 Windows 专属 transitive dep（如 `webview2-com` → 某个使用 `zeroize` 的子 crate）。
3. **本地 Linux 环境**（Rust 1.96.0，target = x86_64-unknown-linux-gnu）走 webkit2gtk 路径，不拉 `zeroize`，所以 `cargo check` 干净通过 → **本地绿，CI 红**。
4. `docs/technical/20-win7-tauri-compat.md` §3.1 列出了 7 个需要 cap 的 crate（hashbrown/ahash/indexmap/ignore/globset/chrono/tokio + uuid + futures-util），**但漏了 zeroize**。zeroize 1.8.x 用 edition2021，1.9.0（2025-08 发布）切到 edition2024。

**修复策略**（3 件套，互锁）：

| # | 改动 | 目的 |
|---|---|---|
| 1 | `Cargo.toml` 加 `zeroize = ">=1.5, <1.9"` cap | 在依赖图根节点阻断 1.9.0+ |
| 2 | 重新生成 `Cargo.lock`（`cargo generate-lockfile`）| 让 lockfile 显式锁住 zeroize 1.8.x（之前 lockfile 不含 zeroize，因 Linux 路径不需要） |
| 3 | CI `Build Tauri` step 加 `CARGO_FLAGS="--locked"` env | 锁住 lockfile，防未来 cargo 自由解析时再次拉新版 |
| 4 | `docs/technical/20-win7-tauri-compat.md` §3.1 + §5 同步补 zeroize 行 | 防回归 |

        # 详见 docs/superpowers/plans/2026-06-13-tauri-2.1.1-zeroize-edition2024-fix.md
